## 0) Project Overview State Flow (ภาพรวม)
Start → Config Load → Market Data/Indicators → Signal Package → AI Confirm → Execution → Position State → Hybrid SL Manager → Logging/Notify → Risk Controls → Completed

**Definition of “Completed” (ต้องมีครบ)**
1) AI response schema + validator (ล็อก output ให้ deterministic)
2) offline test harness (รันซ้ำได้ทุกวัน)
3) regime/event guardrails ที่เป็นกฎตายตัว (ไม่ปล่อยให้ AI เดา)
4) kill-switch policy (แพ้ติด/สเปรดสูง/DD วันเดียว)

## 2) Technical Indicators & Tools Used (Indicators/Tools)

### Indicators (คำนวณ/ใช้เป็นเงื่อนไข)
- ATR(14) → ระยะ SL/TP, proximity, volatility filter
- SuperTrend(10, 3.0) → ทิศทาง/กรองเทรนด์
- Bollinger Bands(20, 2.0) → sideway scalping, touch/near outer band
- ADX(14) → จำกัดภาวะ sideway (ADX ต้องไม่แรงเกิน)
- RSI(14) → overbought/oversold + soft trigger
- Market Structure (HH/HL/LH/LL), BOS/CHoCH → สร้าง bias/สัญญาณ
- Breakout/Retest + proximity scoring
- Sideway bandwidth filter (BB width limit, ATR-based)

### Tools/Infra
- MetaTrader5 Python API: positions_get, history_orders_get, order_send
- Telegram notifier (ปัจจุบัน config ระบุ telegram.enabled=false ในบางช่วง)
- API Server (Flask) สำหรับ preview/test
- Dashboard HTML/Python สำหรับดูสถานะ
- .state JSON สำหรับ persistence ของ initial risk / executor state
