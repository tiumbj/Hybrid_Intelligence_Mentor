3) State Diagram (Wording) — ภาพรวม End-to-End
[*] START

→ CONFIG_RESOLUTION

ใช้ config_resolver.py + config.json + strategy_profiles.py

Output: CFG_EFFECTIVE

→ ENGINE_EVALUATION

engine.py สร้าง decision contract:

direction, entry_candidate, stop_candidate, tp_candidate, rr, score, confidence_py

bos (top-level)

supertrend_ok (top-level)

context{…} (รายละเอียด MTF/HTF/LTF, thresholds, watch_state, blocked_by, etc.)

→ EXECUTION_GATE (Mentor Executor)

mentor_executor.py

อ่าน BOS จาก top-level pkg["bos"] (ไม่อ่านจาก debug context)

ทำ NONE_REASON ให้ boolean เสมอ (เช่น bos=False ไม่เป็น None)

เคารพ ai.enabled:

false → ใช้ technical gate ล้วน

true → AI confirm-only (optional)

→ (แตกแขนง)

STATE: APPROVED

ส่ง Telegram ผ่าน telegram_notifier.py

บันทึกผ่าน trade_logger.py

อัปเดตผ่าน performance_tracker.py

(ถ้ามี execution จริง) จะเข้าส่วน MT5 order safety (ยังต้อง hardening)

STATE: NONE / SKIP

แสดง NONE_REASON (เช่น no_bos, no_proximity, supertrend_conflict, counter_breakout)

จบ cycle (หรือรอรอบถัดไป)

→ END

nested states (ภายใน ENGINE_EVALUATION):

Bias State (HTF/MTF/LTF + fallback)

Breakout State (BOS confirm)

Proximity Watch State (WATCH_PROXIMITY / NONE)

Retest State (retest_ok)

SuperTrend Gate (supertrend_ok)

5) สถานะความสำเร็จ “ล่าสุด” (ตามงานที่คุยกัน)
