State Diagram.txt — Hybrid Intelligence Mentor (HIM)
0) ภาพรวมระบบ (System Overview)

ระบบ HIM แบ่งเป็น 5 ชั้น (Layers) ที่เชื่อมกันเป็น state machine:

Config Layer

โหลด/resolve ค่า config และ strategy profile ให้ engine ใช้เป็น “source-of-truth”

Core Engine Layer (Deterministic)

ดึงข้อมูลราคา (ผ่าน MT5 หรือ Shim)

คำนวณอินดิเคเตอร์/โครงสร้างตลาด

สร้าง signal package (direction/entry/sl/tp/rr/context)

AI Confirm Layer (Bounded / Confirm-only)

AI อ่าน signal package → ตอบ CONFIRM/REJECT + proposed adjustments (bounded)

ต้องผ่าน Schema Validator ก่อนส่งต่อไป execution

Execution Layer (MT5)

ส่งคำสั่ง MT5 ตาม engine + guardrails

มี monitor/close/log

Testing/Validation Layer

Replay runner (MT5Shim injection)

Candidate export (BUY/SELL) + Trace export (ทุกแท่ง)

Analyzer สรุป candidate + blocked_by distribution

(ถัดไป) Forward outcome simulator เพื่อวัด winrate/expectancy

1) Technical Indicators & Tools ที่ใช้ (Indicators/Tools Inventory)
1.1 Indicators (Core)

ATR(14): วัด volatility / proximity / SL distance

ADX(14): filter regime (trend vs sideway)

Bollinger Bands(20,2): filter sideway ด้วย BBWidth + mean reversion zones

RSI(14): confirm momentum/overbought-oversold (ตาม logic ใน engine)

SuperTrend(10,3): trend filter / direction alignment

Structure / BOS / CHoCH: market structure confirmation

(optional helpers) FVG/OB หากถูกใช้ในบาง branch

1.2 Tools/Libs

MetaTrader5 Python API: live data + execution

MT5Shim: replay data provider แทน MT5 ใน offline testing

Python: pandas/numpy (สำหรับ replay/analyzer), json/http (API), logging

Flask (api_server.py): expose endpoints สำหรับ preview/signal/telegram test (ตามโปรเจคเดิม)

Legend: ✅ Done | 🟡 In Progress | ⏳ Pending | 🔴 Issues

A) Config Layer

A1) config.json

Function: เก็บ mode, symbol, risk parameters, strategy profiles/timeframes

Config used: symbol, mode, profiles[mode].timeframes (ltf/mtf/htf), risk floors (RR), indicator params

Current: ต้องสอดคล้องกับ replay/engine และ MT5 symbol (เช่น GOLD)

Status: 🟡 In Progress

Why: ใช้งานได้แล้ว แต่ยังต้อง fine-tune thresholds (เช่น sideway gates, RR floor policy)

A2) config_resolver.py, config.py

Function: โหลด/validate config, fallback defaults, expose resolved config ให้ engine/runner ใช้

Status: ✅ Done (ใช้งานจริง)

Why: pipeline ทำงานต่อเนื่องแล้ว

B) Core Engine Layer (Deterministic)

B1) engine.py

Function หลัก:

ดึง rates ผ่าน MetaTrader5.copy_rates_from_pos(...)

สร้าง multi-timeframe frames (LTF/MTF/HTF)

คำนวณ indicators: ATR/ADX/BB/RSI/SuperTrend + structure/BOS/CHoCH

สร้าง signal_package:

direction (BUY/SELL/NONE)

entry/sl/tp/rr

context.blocked_by (เหตุผลที่ reject)

Config used: timeframes (ltf/mtf/htf), indicator params, min_rr, regime filters

Current number/version: ใช้เป็น core; ในช่วงนี้ “logic gate tuning” เป็นงานหลัก

Status: 🟡 In Progress

Why: Engine run ได้ + no exception แต่คุณภาพ/threshold ต้องวัดด้วย outcome simulator

B2) strategy_profiles.py

Function: นิยาม profile ของ strategy (เช่น sideway_scalp/swing) + params

Status: 🟡 In Progress

Why: ต้อง align กับผล backtest/blocked_by distribution เพื่อปรับ filter

C) AI Confirm Layer

C1) ai_mentor.py

Function: รับ signal package → ตอบ confirm-only (ไม่ override direction/lot/mode)

Config used: AI prompt policy, constraints (tighten-only SL, bounded entry shift, RR floor)

Status: 🟡 In Progress

Why: Spec ชัดแล้ว แต่ยังขาด “Validator enforcement” แบบ deterministic

C2) ai_bridge_v1.py

Function: glue ระหว่าง engine ↔ AI API / mock mode / response parsing

Status: 🟡 In Progress

Why: ต้อง integrate validator ก่อนส่งให้ executor

C3) validator_v1_0.py (NEW)

Function: Validate AI decision schema + hard rules (confirm-only, tighten-only SL, RR floor, bounded entry shift, schema strict)

Config used: min_rr, entry shift bound (ATR/points), confirm_min_conf, tighten-only policy

Current number: v1.0.0 (Spec done) แต่โค้ดยังไม่ได้ทำ

Status: ⏳ Pending

Why: “งานถัดไป” ที่ย้ายไปทำใน chat ใหม่ (I task)

C4) tests/test_validator_v1_0.py (NEW)

5) State Machine (Wording Diagram)
STATE 0: Bootstrap

Input: repo + environment + MT5 terminal running

Transition → ConfigReady เมื่อ config load สำเร็จ

STATE 1: ConfigReady

Files: config.json, config_resolver.py, strategy_profiles.py

Transition → EngineReady เมื่อ engine init ได้ (ไม่มี exception)

STATE 2: EngineReady (Deterministic Core)

File: engine.py

Output: signal_package

Transition → ReplayReady เมื่อ replay runner inject MT5Shim สำเร็จ

STATE 3: ReplayReady (Data Harness)

File: candidate_replay_runner_v1_1_0.py (v1.2.2)

Output:

candidates jsonl (BUY/SELL)

trace jsonl (ALL bars)

Transition → AnalysisReady เมื่อ analyzer รันผ่านและสร้าง summary

STATE 4: AnalysisReady (Filter Behavior Known)

File: analyzer_v1_0.py (v1.0.1)

Output: blocked_by distribution + candidate stats

Transition → AIContractReady เมื่อ validator + tests เสร็จ

STATE 5: AIContractReady (Guardrails Enforced)

Files: validator_v1_0.py, tests/test_validator_v1_0.py

Output: deterministic validation

Transition → IntegrationReady เมื่อ ai_bridge/executor เรียก validator ทุกครั้ง

STATE 6: IntegrationReady (Engine→AI→Validator→Executor)

Files: ai_bridge_v1.py, mentor_executor.py

Output: live dry-run + telegram confirm optional

Transition → OutcomeValidated เมื่อ forward simulator วัด winrate/expectancy ผ่านเกณฑ์

STATE 7: OutcomeValidated (Commissioning Gate Passed)

File: forward_simulator (pending)

Output: winrate/expectancy/DD ผ่าน threshold

Transition → LiveShadow → LiveProduction
